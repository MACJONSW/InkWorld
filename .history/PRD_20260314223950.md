# 墨境（InkWorld）产品需求文档（PRD）

## 1. 文档信息

- 产品名称：墨境（InkWorld）
- 文档类型：现状需求文档（As-Is PRD）
- 文档版本：v1.0
- 基线分支：main
- 更新时间：2026-03-14
- 适用范围：Web 端单体应用（Flask + SQLite + 原生前端）

## 2. 产品概述

墨境是一个面向长篇小说创作的 AI 辅助写作工作台，目标是把“写作工程能力（目录、版本、导入导出）”与“叙事一致性能力（多智能体、三层记忆、人物提醒、世界状态）”合并在同一界面中，支持从灵感规划到正文生成、校验、沉淀和迁移的闭环。

产品核心价值：

1. 提升长篇写作效率。
2. 降低角色崩坏、设定冲突、时间线混乱等一致性风险。
3. 支持长期项目资产化管理（设定、摘要、伏笔、心理档案、状态记录）。

## 3. 产品目标与非目标

### 3.1 产品目标

1. 提供完整的“规划-创作-校验-沉淀-导出”工作流。
2. 支持按用户隔离的模型配置、任务路由和生成参数。
3. 支持章节级持续创作与历史可追溯（版本分支 + Diff）。
4. 提供可解释的记忆辅助：三层记忆、人物提醒、向量检索。
5. 支持工作区级迁移（JSON 导入导出）。

### 3.2 非目标（当前版本不承诺）

1. 多人实时协同编辑。
2. 分布式高并发 SaaS 级部署。
3. 移动端独立 App。
4. 企业级密钥托管与细粒度 RBAC。

## 4. 目标用户与典型场景

### 4.1 目标用户

1. 长篇作者：网文、类型小说、连载作者。
2. 剧情策划型作者：重视角色弧线、伏笔回收和世界一致性。
3. 技术型创作者：需要可配置模型路由与可扩展 Agent 管线。

### 4.2 典型场景

1. 从灵感生成卷章大纲并拆解场景节拍。
2. 在章节中流式创作并使用续写/改写/扩写指令。
3. 在完章后生成摘要，自动沉淀人物历史并刷新提醒。
4. 在连载中持续校验世界状态、伏笔、心理和潜台词。
5. 将整本书导出为 Markdown/TXT/EPUB/JSON，或从 JSON 导入。

## 5. 业务流程（端到端）

### 5.1 首次使用流程

1. 用户注册或登录。
2. 进入设置中心添加模型配置。
3. 配置任务路由与生成参数。
4. 新建书籍并建立目录树。

### 5.2 创作主流程

1. 使用架构师生成大纲。
2. 使用节拍器生成场景节拍。
3. 在编辑器中写作或调用执笔者/续写。
4. 使用验证者、幻觉检测、世界状态验证做一致性检查。
5. 使用摘要器沉淀章节摘要并刷新人物历史。
6. 通过记忆面板进行向量检索与实体查询。

### 5.3 资产化与交付流程

1. 管理 Lorebook、实体关系、伏笔池、心理档案、世界状态。
2. 维护版本分支并进行 Diff 对比。
3. 导出成外部格式或导入 JSON 工作区继续创作。

## 6. 信息架构与页面结构

### 6.1 页面总览

1. 顶部栏：书籍切换、字数/Token、导出导入、专注模式、设置、账户。
2. 左栏：目录树、Lorebook、实体图谱。
3. 中栏：编辑器、Slash 指令、Ghost Text、版本与 Diff、流式状态。
4. 右栏：智能体工作台、记忆面板（记忆状态、人物提醒、张力诊断、摘要、检索、查询）。

### 6.2 弹窗与系统面板

1. 设置中心：模型管理、任务路由、生成参数、Token 统计。
2. 新建书籍弹窗。
3. Lorebook 编辑弹窗。
4. 导出菜单弹窗。
5. Diff 对比弹窗。
6. 实体关系编辑弹窗。
7. 认证弹窗。

## 7. 功能需求

### 7.1 认证与用户隔离

- FR-AUTH-001：支持邮箱注册、登录、获取当前用户信息。
- FR-AUTH-002：API 需 Bearer JWT 鉴权（白名单接口除外）。
- FR-AUTH-003：每个用户的数据必须隔离，包括书籍、模型、路由、参数、Token 统计。
- FR-AUTH-004：JWT 过期后前端需提示重新登录。

验收标准：

1. 未登录访问受保护接口返回 401。
2. A 用户不能读取或操作 B 用户书籍/节点。
3. 登录后刷新页面仍可恢复会话（本地 token）。

### 7.2 模型中心与任务路由

- FR-MODEL-001：支持模型配置 CRUD（名称、提供商、Base URL、API Key、模型 ID、最大上下文）。
- FR-MODEL-002：API Key 需加密存储，前端列表仅显示脱敏值。
- FR-MODEL-003：支持角色级路由映射（planner、drafter、validator 等）。
- FR-MODEL-004：支持用户级生成参数设置（temperature、top_p、penalty、max_tokens）。

验收标准：

1. 未配置某角色时可回退默认模型。
2. 路由修改后，后续该角色调用使用新模型。
3. 生成参数变更后对后续 LLM 请求生效。

### 7.3 书籍与目录树管理

- FR-BOOK-001：支持书籍 CRUD（标题、简介、作者、类型）。
- FR-BOOK-002：支持节点管理（卷/章/场景），并形成树结构。
- FR-BOOK-003：支持拖拽排序与父子重排持久化。
- FR-BOOK-004：节点可维护状态（草稿/定稿）。

验收标准：

1. 可创建多级目录并正确渲染。
2. 拖拽后刷新页面，顺序保持一致。
3. 删除节点应级联删除子节点及其内容。

### 7.4 编辑器与写作交互

- FR-EDIT-001：提供 contenteditable 写作区和自动保存（防抖）。
- FR-EDIT-002：实时字数统计与顶部联动显示。
- FR-EDIT-003：支持 Slash 指令入口和划词快捷操作。
- FR-EDIT-004：支持 Ghost Text 自动补全（Tab 接受、Esc 取消）。
- FR-EDIT-005：支持专注模式（隐藏侧栏，纯净写作）。
- FR-EDIT-006：支持流式生成状态显示与手动停止生成。

验收标准：

1. 编辑 2 秒后自动保存状态变为已保存。
2. Slash 指令能触发对应能力。
3. 自动补全可正确插入光标位置。
4. 专注模式桌面端可无干扰编辑。

### 7.5 版本分支与 Diff

- FR-VERSION-001：节点支持多版本分支（A/B/C 等）。
- FR-VERSION-002：支持激活某分支并覆盖当前内容。
- FR-VERSION-003：支持任意两版本 Diff 对比。
- FR-VERSION-004：支持“接受新版本”写回编辑器。

验收标准：

1. 同一节点可创建多个版本并切换。
2. Diff 可显示增删改逐行结果。
3. 应用后内容可自动保存。

### 7.6 多智能体能力（功能矩阵）

当前内置能力如下：

1. 架构师（Planner）：整书/整卷大纲。
2. 节拍器（Beat Generator）：章节节拍拆解。
3. 执笔者（Drafter）：流式正文创作。
4. 验证者（Validator）：OOC、时间线、设定一致性检查。
5. 润色器（Polisher）：风格化润色。
6. 摘要器（Summarizer）：章节结构化摘要与角色提及提取。
7. 智能续写（Continuation）：批评重试 / 快速续写 / Plan 模式入口。
8. 自动补全（Autocomplete）：低延迟 Ghost Text。
9. 冲突设计（Conflict Design）：对抗矩阵与 A/B/C 冲突方案。
10. 联想风暴（Association）：因果/反转/细节多探针创意卡。
11. 伏笔检测与待填坑扫描。
12. 潜台词分析（双轨分析）。
13. 心理透视（角色深层心理解读）。
14. 世界状态提取与一致性验证。
15. Plan-and-Solve 深度生成（三阶段）。
16. 幻觉检测（NLI + 世界状态融合）与执笔自动防护（重试上限 3）。
17. 行内编辑指令（续写/改写/扩写环境/精简对话/增加张力/内心独白）。

通用需求：

- FR-AGENT-001：所有 Agent 应支持用户上下文隔离。
- FR-AGENT-002：流式 Agent 使用 SSE 输出，结束标记 [DONE]。
- FR-AGENT-003：可通过 stop 接口中断当前生成。
- FR-AGENT-004：记录按角色维度的 Token 消耗统计。

验收标准：

1. 各 Agent 入口在 UI 中可访问并可返回结果。
2. 流式输出逐步更新，不阻塞主线程。
3. 停止后应尽快终止流。
4. 统计面板可查看角色维度累计 Token。

### 7.7 三层记忆系统

- FR-MEM-001：Tier1 工作记忆，基于当前节点内容截取最新文本。
- FR-MEM-002：Tier2 滚动摘要，基于章节摘要形成前情窗口。
- FR-MEM-003：Tier3 向量记忆，支持 TF-IDF，优先 FAISS，融合 BM25（可选）。
- FR-MEM-004：支持整书向量化构建、Top-K 检索。
- FR-MEM-005：支持动态设定注入（关键词/向量相似度混合）。

验收标准：

1. 记忆状态面板可显示三层状态。
2. 构建索引后可检索出相关段落。
3. 无索引时应自动触发构建或返回空结果，不崩溃。

### 7.8 人物提醒与人物历史

- FR-CHAR-001：自动识别当前文本提及角色（名称 + 关键词）。
- FR-CHAR-002：聚合角色人格、近期历史、未回收伏笔、世界状态。
- FR-CHAR-003：提醒同时用于右侧展示与提示词注入。
- FR-CHAR-004：支持手工新增、编辑、删除人物历史。
- FR-CHAR-005：支持章节刷新与全书回填。
- FR-CHAR-006：摘要后可触发人物历史自动更新。

验收标准：

1. 提及角色后提醒面板出现对应卡片。
2. 手工记录可立即体现在提醒卡。
3. 回填全书后返回章节数与新增条目数。

### 7.9 世界观资产管理

- FR-ASSET-001：Lorebook 支持角色/地点/物品/派系/法则分类。
- FR-ASSET-002：实体关系图谱支持 source-target-type-value。
- FR-ASSET-003：伏笔池支持增删改查、状态切换与填坑记录。
- FR-ASSET-004：世界状态支持按实体和类型维护。
- FR-ASSET-005：心理档案支持角色维度维护。

验收标准：

1. 资产 CRUD 操作成功后列表及时刷新。
2. 状态/关系可用于后续查询与校验。
3. 伏笔可从未填切换为已填并记录回收信息。

### 7.10 查询、检索与诊断

- FR-DIAG-001：划词查询返回相关设定、关系、世界状态。
- FR-DIAG-002：提供情绪张力曲线诊断（章节分值、预警）。
- FR-DIAG-003：提供章节摘要列表与一键生成。
- FR-DIAG-004：提供实体与向量检索双入口。

验收标准：

1. 选择文本可触发查询并展示结果。
2. 张力诊断可返回均值、警告和章节曲线。
3. 摘要生成后可在列表看到新记录。

### 7.11 导入导出

- FR-IO-001：导出 Markdown。
- FR-IO-002：导出 TXT。
- FR-IO-003：导出 EPUB（无依赖时回退 HTML）。
- FR-IO-004：导出 JSON 工作区。
- FR-IO-005：导入 JSON 工作区并重建映射关系。

验收标准：

1. 四种格式均可下载。
2. 导入后书籍、目录、内容、版本、设定、摘要、伏笔、状态、心理、人物历史可恢复。
3. 节点与分支 ID 映射正确，无跨书污染。

## 9. 数据需求与核心对象

### 9.1 核心对象

1. User：账号主体。
2. Model：模型配置（加密 API Key）。
3. UserRouting：角色到模型映射。
4. UserGenerationParams：用户生成参数。
5. Book：书籍元数据。
6. Node：目录树节点（卷/章/场景）。
7. NodeContent：节点当前内容。
8. Version：节点分支内容。
9. LorebookEntry：世界观条目。
10. EntityRelation：实体关系。
11. ChapterSummary：章节摘要。
12. Outline：大纲文本。
13. Foreshadowing：伏笔条目与状态。
14. WorldState：世界状态。
15. CharacterPsychology：角色心理档案。
16. CharacterHistory：角色历史事件。
17. TokenStat：调用统计。

### 9.2 数据关系

1. User 1:N Book。
2. Book 1:N Node（树形）。
3. Node 1:1 NodeContent。
4. Node 1:N Version。
5. Book 1:N Lorebook/Summary/Outline/Foreshadowing/WorldState/Psychology/CharacterHistory。

### 9.3 数据一致性原则

1. 所有业务写入应绑定 user_id 或通过所属 book_id 间接绑定用户。
2. 节点操作需校验 node_id 与 book_id 一致。
3. 删除上层对象需级联清理下游数据。
4. 导入场景需进行旧 ID 到新 ID 映射。

## 10. 非功能需求

### 10.1 安全与隐私

1. JWT 鉴权，默认过期 30 天。
2. API Key 与敏感文本使用本地 Fernet 加密。
3. 书籍/节点级访问控制防止跨用户访问。
4. 默认开发密钥仅用于本地测试，生产需替换。

### 10.2 性能与体验

1. 核心创作链路支持 SSE 流式，降低首字延迟体感。
2. 自动保存防抖，减少无效写入。
3. 自动补全请求可中断，避免并发堆积。
4. 向量检索支持 FAISS 可选加速。

### 10.3 可靠性

1. SQLite 使用 WAL 与外键约束。
2. Optional 依赖缺失时需可降级（如 BM25/FAISS/jieba）。
3. 导入导出流程需保证最小可恢复集（book/tree/content）。

### 10.4 可扩展性

1. 新 Agent 可按统一模式接入（agents + app + 前端面板）。
2. 模型路由按角色扩展，无需改动核心路由协议。
3. 记忆能力可通过 MemoryEngine 扩展并接入提示词。

### 10.5 可用性

1. 主要能力应在单页内可达，减少跳转。
2. 所有关键操作有可见反馈（toast、状态栏、加载态、警告栏）。
3. 专注模式支持长时间纯写作场景。

## 11. 异常处理与失败策略

1. 鉴权失败：返回 401，前端清 token 并提示重新登录。
2. 模型未配置：返回可读错误文本，提示先配置路由。
3. 流式异常：终止流并给出错误提示。
4. 幻觉防护失败：标记未通过，给出冲突列表并阻断直接上屏。
5. 可选依赖缺失：回退到基础检索路径，不导致系统不可用。

## 12. 验收标准（UAT）

### 12.1 核心链路验收

1. 从注册到完成第一章写作并导出 Markdown 全流程可执行。
2. 在同一本书中完成“规划-节拍-执笔-验证-摘要”闭环。
3. 人物提醒可在提及角色时给出历史与伏笔提示。

### 12.2 质量与一致性验收

1. 幻觉检测可识别至少一种显式冲突并返回解释。
2. 世界状态提取后，验证接口可发现状态矛盾。
3. 伏笔检测与伏笔池联动可完成新增和回收。

### 12.3 工程能力验收

1. 版本分支可创建、切换、对比并应用。
2. JSON 导出后重新导入可恢复主要资产。
3. Token 统计面板可按角色查看累计值。

## 13. 当前实现边界与限制

1. 角色识别主要依赖角色名与关键词，别名需人工补入 keywords。
2. Token 统计在部分流式链路使用估算值，非精确计费口径。
3. SQLite + 本地密钥机制更适合单机或小规模环境。
4. 前端布局以桌面端为主，移动端仅有限适配。
5. 部分高级能力质量高度依赖所绑定模型能力。

## 14. 版本外建议（不属于当前验收范围）

1. 增加协同写作与冲突合并机制。
2. 增加更强别名识别与实体消歧。
3. 增加审计日志、权限分级与云端密钥托管。
4. 增加可观测性指标（首字延迟、重试率、一致性命中率）。
